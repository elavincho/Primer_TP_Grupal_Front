> Bitácora del Proyecto_

Registro del proceso de desarrollo del TP1 - Proyecto Web en Equipo

> Decisiones de Diseño_

  • Estética Terminal/Hacker: Decidimos adoptar una estética de terminal de comandos con colores verde neón (#00ff41) sobre fondo oscuro (#0a0a0a) para darle un aspecto técnico y moderno al proyecto.
  • Tipografía: Utilizamos la fuente "JetBrains Mono" de Google Fonts, una fuente monoespaciada que refuerza la estética de terminal y mejora la legibilidad del código.
  • Estructura de Navegación: Implementamos un header consistente en todas las páginas con el logo "404_SOLUTIONS_" y enlaces a Inicio y Bitácora, asegurando que el usuario nunca necesite usar el botón "Atrás" del navegador.
  • Páginas Individuales: Cada integrante tiene libertad creativa en su página personal, manteniendo la coherencia en header y footer pero permitiendo estilos únicos en el contenido principal.
  • Responsive Design: Implementamos breakpoints en 400px, 900px y 1200px para asegurar una experiencia óptima en móviles, tablets y escritorio.

> Dificultades Encontradas y Soluciones_

  • Problema: Inconsistencias en el header entre diferentes páginas.
    Solución: Estandarizamos la estructura HTML del header y creamos estilos CSS específicos para asegurar que el logo y los enlaces tengan el mismo color (#00ff41) en todas las páginas.
  • Problema: El footer de una página tenía un diseño complejo diferente al resto.
    Solución: Simplificamos el footer para que todas las páginas tengan la misma estructura: "© [año] IFTS N.°29 - 404 Solutions" con año dinámico mediante JavaScript.
  • Problema: Estructura HTML incorrecta con elementos fuera del body.
    Solución: Reorganizamos la estructura HTML para que todos los elementos visibles estén dentro del <body> y seguir las buenas prácticas de HTML5.
  • Problema: Organización de archivos CSS y JS.
    Solución: Creamos carpetas separadas (css/, js/, img/) y organizamos todos los recursos de manera clara y estructurada.
  • Problema: Funcionalidad de párrafos expandibles en las descripciones de películas.
    Solución: Implementamos JavaScript con event listeners para alternar entre estados "resumido" y "completo" mediante clases CSS.

> Cambios Importantes Durante la Implementación_

  • Unificación del Header: Inicialmente cada página tenía un header ligeramente diferente. Se estandarizó la estructura y los colores para mantener consistencia visual.
  • Simplificación del Footer: Se eliminó un footer complejo con redes sociales y se adoptó un footer minimalista uniforme en todas las páginas.
  • Botones de Navegación: Se agregaron botones "_VOLVER_AL_INICIO" en todas las páginas individuales para cumplir con el requerimiento de navegación sin usar el botón "Atrás" del navegador.
  • Año Dinámico: Se implementó JavaScript para actualizar automáticamente el año en el footer, evitando tener que modificar manualmente el código cada año.
  • Accesibilidad: Se agregaron atributos ARIA (role="banner", role="contentinfo", aria-label) para mejorar la accesibilidad del sitio.
  • Interactividad JavaScript: Se implementaron funciones dinámicas como:
    • Botón de diagnóstico en la portada con alert
    • Párrafos expandibles en descripciones de películas
    • Actualización dinámica del año en el footer

> Tecnologías y Herramientas Utilizadas_

  • HTML5: Estructura semántica del sitio
  • CSS3: Estilos, animaciones y diseño responsive
  • JavaScript: Interactividad y funciones dinámicas
  • Google Fonts: Tipografía JetBrains Mono
  • Git/GitHub: Control de versiones y colaboración
  • Vercel: Despliegue y hosting del proyecto
  • IA (Gemini/Claude): Asistencia en debugging, optimización de código y resolución de problemas técnicos
